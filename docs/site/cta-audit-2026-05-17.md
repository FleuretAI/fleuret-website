# CTA audit — fleuret.ai (2026-05-17)

> Source: W2 of the YC website playbook backlog (see `TODOS.md`). Audit-only doc, NO code changes in this PR. Recommendations land in a follow-up PR per page, after Yanis sign-off.

## Goal

Each top page should drive **one** directional next-step CTA matching that page's buyer intent. Today the site has two systemic problems:

1. **Demo as the only verb.** Almost every CTA = "Book a demo → /demo". Pages with a different natural intent (read, apply, share, learn) still funnel to /demo, splitting reader attention instead of capturing the actual next step they'd take.
2. **Tracking blind spots.** Of 18 CTAs, only 5 fire `trackCTAClick`. The 5 tracked are all demo-funnel from Hero + Navbar + CTASection. Every secondary verb (career, blog, resources, footer email, AI provider link) is untracked, so conversion data is half-blind.

## Method

- Read every page in `src/pages/` + every shared component composed into them.
- Catalog visible Link / button / anchor that drives a navigation event.
- Skip pure scroll-cues (e.g. Hero `scrollCue` chevron).
- Skip legal/admin pages (PrivacyPolicy, MentionsLegales, SecurityPolicy, SubProcessors, TermsOfUse, NotFound) — no growth surface.

## Findings

### F1 — Two demo CTAs on the pricing card row (`PricingSection.tsx:158` + `:316`)
Standard tier says "Book a demo", Enterprise inline says "Contact us →". Same destination. Two competing labels for the same action = decision fatigue + zero differentiation. Pick one verb across the price section.

### F2 — `/design-partners` hero has 2 CTAs of different intents
"Book a demo" (general funnel) competes with "See how it runs" (in-page scroll). Page is specifically a design-partners apply page with form at `#apply`. Hero should drive **to the form**, not to /demo. Demo prospect is the wrong buyer for this page.

### F3 — Blog has no CTA at all
Visitor reads 1500 words, hits the end, sees only "Share on LinkedIn / Copy URL". No newsletter capture, no demo, no related post. **This is W4's actual problem surfacing here.** W2 fix = decide what the per-blog-post next-step is. W4 will implement it.

### F4 — Career page sends `mailto:` only
No applicant tracking. No structured form. `yanis@fleuret.ai` is the public CEO email. Real applicants → mixed with sales/investor email. Recommendation: keep mailto for now, but add a tracked event on click so volume is measurable.

### F5 — Tracking blackout on 13 of 18 CTAs
Demo-funnel from Hero / Navbar / CTASection tracks. Everything else doesn't. No way to attribute conversions from /compliance/* pSEO pages, /design-partners apply form, /about, /careers, /resources, blog. **GA4 funnel = blind below the first demo click.** Coordinate with `feat/ga4-debug-mode` sibling agent before broad tracking work (see W7).

### F6 — "Contact us →" arrow chevron is the only directional cue in the price section
Hero / CTASection use the same arrow inconsistently. Adopt one pattern repo-wide: arrow only on `cta-tertiary` (text-link) variant, never on primary buttons.
