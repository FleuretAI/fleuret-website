### Fixed

- **`/changelog` rendered blank in production (`TypeError: t.split is not a function`).** `@mdx-js/rollup` defaults to processing both `.mdx` and `.md` files, so it intercepted `CHANGELOG.md` before Vite's `?raw` query handler could return the raw source string. The bundled chunk shipped a JSX component instead of a string, and `parseChangelog(raw)` crashed on the first `.split()`. Fixed in `vite.config.ts` by passing `mdExtensions: []` to `mdx()`, restricting MDX to `.mdx` files only. All blog posts are `.mdx`, so this has no impact on existing routes; it just frees `.md` for raw imports. Verified by inspecting the post-build chunk (`const A='# Changelog\\n...'` instead of an MDX component function).

### Fixed

- **CSP blocked GA4 Consent Mode v2 pings to `https://www.google.com/g/collect`.** `connect-src` in `vercel.json` (mirrored in `nginx.conf`) allowed `*.google-analytics.com`, `*.analytics.google.com`, and `stats.g.doubleclick.net` but not `www.google.com`, so every pageview emitted a CSP violation in the console and EU Consent Mode v2 signals never reached Google. Added `https://www.google.com` to the `connect-src` directive in both configs. No other directives changed.
- **GA4 consent never upgraded to granted, SPA navigations never tracked.** `src/components/CookieBanner.tsx` dispatched a `cookie-consent` CustomEvent on accept/deny, but nothing listened, so every visitor stayed stuck in Consent Mode v2 `denied` and GA4 only received anonymized cookieless pings (no attribution, no returning-visitor data, sparse Insights). On top of that, React Router route changes never fired `page_view`, so only the initial HTML load was counted. Added `src/components/Analytics.tsx` mounted inside `BrowserRouter` that (1) listens to `cookie-consent` and fires `gtag('consent','update',{analytics_storage:'granted'|'denied'})`, (2) replays stored `localStorage` consent on mount so returning visitors who already accepted are re-granted, (3) emits `gtag('event','page_view',...)` on every route change (skipping the first render so the initial `gtag config` page_view is not double-counted). Extended `src/lib/gtag.ts` with typed `grantAnalyticsConsent`, `denyAnalyticsConsent`, and `trackPageView` helpers that no-op under SSR / prerender / blocked-gtag. Stale consent-denied comment block in `index.html` updated to describe the bridge. Separately, GA4 â†” Search Console link must still be configured in the GA4 admin UI to populate the "Web Search" dimension in Insights â€” that's a Google admin action, not a code change.
